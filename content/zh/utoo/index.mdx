# utoo

import { Callout, Cards, Tabs } from 'nextra/components'

utoo (`ut`) 是一个快速、现代的 JavaScript 包管理器，提供与 npm 兼容的包管理功能，并具有性能优化。

<Callout type="info">
  utoo 是 npm 的直接替代品，性能更优。
</Callout>

## 安装

<Tabs items={['macOS / Linux', 'Windows', 'npm']}>
  <Tabs.Tab>
    无需 Node.js，一行命令安装：

    ```bash filename="Terminal" copy
    curl -fsSL https://utoo.land/install | bash
    ```
  </Tabs.Tab>
  <Tabs.Tab>
    在 PowerShell 中运行：

    ```powershell filename="PowerShell" copy
    irm https://utoo.land/install.ps1 | iex
    ```
  </Tabs.Tab>
  <Tabs.Tab>
    如果你已经安装了 Node.js：

    ```bash filename="Terminal" copy
    npm install -g utoo
    ```
  </Tabs.Tab>
</Tabs>

## 快速开始

### 1. 安装依赖

```bash filename="Terminal" copy
ut install
```

### 2. 添加包

```bash filename="Terminal" copy
ut install lodash
ut install -D typescript
```

### 3. 运行脚本

```bash filename="Terminal" copy
ut build
ut dev
```

### 4. 执行远程包

```bash filename="Terminal" copy
utx create-react-app my-app
```

## 特性

### 全局缓存

安装过的包会按 `包名/版本` 缓存到 `~/.cache/nm/`，后续安装直接从缓存硬链接到项目，跳过下载和解压。同一版本的包在磁盘上只存储一份，跨项目共享。

```
~/.cache/nm/
├── lodash/4.17.21.tgz
├── react/18.3.0.tgz
└── typescript/5.6.0.tgz
```

### npmmirror 加速

使用 npmmirror 作为默认镜像源时，utoo 提供两项额外加速：

- **Binary 镜像** — 原生模块（如 `esbuild`、`swc`、`cypress`）的预编译二进制从 npmmirror CDN 下载，无需访问 GitHub Releases 等海外源
- **Semver 请求** — 支持在 URL 中直接传递版本范围（如 `/lodash/^4`），由镜像源服务端解析，减少客户端请求数量

```bash filename="Terminal" copy
# 检测镜像源是否支持 semver 请求
ut ping
# → PONG 15ms (supports-semver: yes)
```

### 从 pnpm 迁移

一行命令从 pnpm 项目迁移，自动转换 `pnpm-workspace.yaml` 中的工作区、overrides 和 catalog 配置：

```bash filename="Terminal" copy
ut install --from pnpm
```

详见[从 pnpm 迁移](/zh/docs/utoo/migration)。

## 快捷执行

<Callout>
  你可以直接运行 package.json 脚本，无需 `run`：
</Callout>

```bash filename="Terminal" copy
ut build      # 等同于: ut run build
ut dev        # 等同于: ut run dev
ut test       # 等同于: ut run test
```
